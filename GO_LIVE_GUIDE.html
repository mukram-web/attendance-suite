AttendaSync · Setup Guide

Make your app read live from Google Drive

Right now AttendaSync needs you to upload three files by hand. This guide hires a little robot helper that fetches your roster, schedule, and Zoom data straight from Drive — so the app stays up to date on its own.

You do not need to know how to code. Every step is a click in a web page or a value you copy and paste. Take it one numbered step at a time — there are four. Set aside about 30–40 minutes the first time.

What you'll do

  1. Create the robot helper in Google Cloud ~15 min
  2. Share your 3 Drive items with the robot ~10 min
  3. Paste the key & file IDs into Streamlit ~10 min
  4. Deploy, then confirm it's live ~5 min
Robot

The big idea, in one breath. Google won't let your app log in as you. Instead you create a service account — a robot Google login with its own email address and its own password-file (a JSON key). You give that key to Streamlit, and you share your three Drive items with the robot's email, exactly like sharing with a colleague. The robot can then read those three items and nothing else.

1

Create the robot helper in Google Cloud

Make a project, switch on two Google services, create the robot, and download its key.

  1. Sign in to the Google Cloud Console.

    Go to console.cloud.google.com and sign in with your normal Google account. This is a free control panel — you will not be charged for what we do here.

  2. Create a project.

    Click the Menu ☰ (top-left) → IAM & Admin → Create a Project (or use the project picker at the top and click NEW PROJECT). Type a Project name like AttendaSync Drive. For Location, a personal Gmail account will simply show "No organization" — that's fine. Click Create and wait a minute for it to finish.

  3. Turn on the Google Drive API.

    Make sure your new project is selected in the picker at the top. Then open this link in the same browser tab:

    console.cloud.google.com/apis/library/drive.googleapis.com → click the blue Enable button.

  4. Turn on the Google Sheets API too.

    Your roster and L2 schedule are real Google Sheets, so the app needs this one as well:

    console.cloud.google.com/apis/library/sheets.googleapis.com → click Enable.

  5. Create the service account (your robot).

    Go to Menu ☰ → IAM & Admin → Service Accounts, then click + CREATE SERVICE ACCOUNT. Give it a name like attendasync-reader and click CREATE AND CONTINUE.

  6. Skip the two optional steps.

    The next screen asks to "grant access to project" (a role). You do not need one — the robot will get access by file-sharing instead. Just click CONTINUE, then DONE. Your robot now appears in the list.

  7. Copy the robot's email address.

    In the Service Accounts list, copy the value in the Email column. It looks like attendasync-reader@attendasync-drive.iam.gserviceaccount.com. Keep this somewhere handy — you'll paste it three times in Step 2.

  8. Download the key file (JSON).

    Click the robot's email to open it → the KEYS tab → ADD KEY → Create new key → choose JSON → CREATE. A .json file downloads to your computer. You'll open it in Step 3.

One copy

That JSON file is downloaded only once and can never be downloaded again. Treat it like a password: don't email it, don't put it in a shared folder, and never upload it to GitHub. If you ever lose it, just come back to the KEYS tab, delete the old key, and create a fresh one.

Gotcha

If the "Create new key" button is blocked with a message about a policy (Key creation is not allowed on this service account), your Google account is a managed Workspace/organization account. Google blocks key creation by default for organizations created on or after 3 May 2024. A free personal Gmail account is not affected. If you hit this, an admin of your organization has to set the policy iam.disableServiceAccountKeyCreation to "Not enforced" — the easiest fix is often to do this whole step with a personal Gmail account instead.

2

Share your 3 Drive items with the robot

The robot starts with an empty Drive. It can only see what you explicitly share with it.

You have three things in Drive that AttendaSync needs. For each one, you'll do two tiny jobs: (A) share it with the robot's email, and (B) grab its file ID from the address bar.

Drive itemWhat it isGoes into the setting
Master RosterGoogle Sheetroster_id
L2 Weekly SessionsGoogle Sheetl2_id
Zoom Attendee Reportsa single .zip fileattendee_zip_id

A · Share each item

  1. Open the item in Google Drive and click the Share button (top-right). For the Sheets you can open the sheet itself; for the .zip file, right-click it in Drive and choose Share.
  2. Paste the robot's email into the "Add people" box (the address you saved in Step 1).
  3. Set the role to Viewer in the dropdown next to it. Viewer (read-only) is all the robot needs.
  4. Untick "Notify people," then click Share.

    The robot has no inbox, so there's no point emailing it. Repeat this for all three items.

Tip

If all three items live in one Drive folder, you can share the folder once and every item inside inherits the access — sharing cascades to the contents. You still need each item's individual file ID below.

B · Find each file's ID in its URL

Open the item in your browser and look at the address bar. The ID is the long random string after /d/ (for Sheets) or after ?id= / inside the link (for the zip). Copy only that string — not the slashes around it.

A Google Sheet URL

https://docs.google.com/spreadsheets/d/1A2b3C4d5E6f7G8h9IjK0LmN0pQrStUvWxYz/edit#gid=0 ↑ copy this highlighted part — that's your roster_id or l2_id

A Drive file (the .zip) — open it, then look at the address bar

https://drive.google.com/file/d/1Zx9Yw8Vu7Ts6Rq5Po4Nm3Lk2Ji1Hg0Fe/view ↑ this highlighted part is your attendee_zip_id
Gotcha

The robot sees the file by its ID, not by its name. When you replace the Zoom .zip after each session, upload a new version of the same file (in Drive: right-click → "Manage versions" → upload new version) so the ID stays the same. If you instead delete it and upload a brand-new file, you'll get a new ID and must update attendee_zip_id in Step 3 — and re-share the new file with the robot.

3

Paste the key & file IDs into Streamlit

This is where the JSON key from Step 1 and the three IDs from Step 2 come together.

Streamlit keeps passwords in a private box called Secrets, written in a simple format called TOML. You'll paste one block of text. Below is the exact template — fill in the highlighted parts.

Secrets · paste into Streamlit TOML
# ── Your robot's key: copy each value from the downloaded JSON file ──
[gcp_service_account]
type                        = "service_account"
project_id                  = "attendasync-drive"
private_key_id              = "abcdef0123456789abcdef0123456789abcdef01"
private_key                 = """-----BEGIN PRIVATE KEY-----
MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcwggSjAgEAAoIBAQ...
...many lines of letters and numbers from your JSON...
-----END PRIVATE KEY-----
"""
client_email                = "attendasync-reader@attendasync-drive.iam.gserviceaccount.com"
client_id                   = "123456789012345678901"
auth_uri                    = "https://accounts.google.com/o/oauth2/auth"
token_uri                   = "https://oauth2.googleapis.com/token"
auth_provider_x509_cert_url = "https://www.googleapis.com/oauth2/v1/certs"
client_x509_cert_url        = "https://www.googleapis.com/robot/v1/metadata/x509/attendasync-reader%40attendasync-drive.iam.gserviceaccount.com"

# ── Your three Drive file IDs from Step 2 ──
[drive]
roster_id       = "1A2b3C4d5E6f7G8h9IjK0LmN0pQrStUvWxYz"   # Master Roster Sheet
l2_id           = "1B3c4D5e6F7g8H9i0JkL1MnO2pQrStUvWxYz"   # L2 schedule Sheet
attendee_zip_id = "1Zx9Yw8Vu7Ts6Rq5Po4Nm3Lk2Ji1Hg0Fe"     # the Zoom .zip

How to fill it in: open the downloaded .json file in any text editor (Notepad works). Each line in the JSON has a name and a value in quotes — copy the value across to the matching line above. The names line up one-to-one.

#1 trap

The private_key line breaks almost everyone the first time. Inside the JSON file it's one very long line full of \n symbols. You have two safe ways to paste it:

Easiest: use the triple-quote style shown above (""" … """) and paste the key as real lines — wherever you see \n in the JSON, press Enter to make a real line break instead. Start with -----BEGIN PRIVATE KEY----- and end with -----END PRIVATE KEY-----.

Or: keep it on one line in normal double-quotes and leave every \n exactly as it is — one backslash, one "n" — like "-----BEGIN PRIVATE KEY-----\nMIIE...\n-----END PRIVATE KEY-----\n".

Gotcha

Do not double the backslashes (\\n is wrong) and do not use single quotes '…' for the one-line style. Either mistake makes Google reject the key with errors like No key could be detected or Could not deserialize key data. Keep exactly one \n in the one-line form, or use the triple-quote real-line form.

Where to paste it:

  1. If you're deploying for the first time: in the Streamlit deploy dialog (next step), click Advanced settings and paste the whole block into the Secrets box.
  2. If the app is already deployed: go to share.streamlit.io, find your app, click the ⋮ menu → Settings → Secrets tab, paste the block, and click Save.
Robot

These secrets live only inside Streamlit, never in your GitHub repo. That's exactly right — the key is a password and must never be committed to a public repo. You're pasting it straight into Streamlit's private box on purpose.

4

Deploy, then confirm it's live

Publish the app (or reboot it) and check that it pulls your three items from Drive.

  1. Open your Streamlit workspace.

    Go to share.streamlit.io and sign in with GitHub.

  2. First time only — deploy the app.

    Click Create app → "Yup, I have an app" → choose your repository and branch, and set the Main file path to app.py. Open Advanced settings, paste your Secrets from Step 3, click Save, then Deploy.

  3. Already deployed — just reboot it.

    After saving new or changed secrets, the app picks them up on its next run. To be sure, open the app's ⋮ menu in your workspace and choose Reboot.

  4. Open the app and watch it load.

    Visit your your-app.streamlit.app URL. Opening or refreshing the page runs the app fresh from the top — so it re-fetches your roster, schedule, and zip from Drive each time you open it. That's your "live" behaviour.

✓ You're live when all of these are true

  • ✓ The app opens without a red error box about credentials or files.
  • ✓ It shows data from your current Drive roster and L2 schedule — not last week's upload.
  • ✓ After you replace the Zoom .zip in Drive and refresh the app, the new session appears.
Good to know

If nobody opens the app for 12 hours, Streamlit puts it to sleep to save resources. The next visitor just clicks "Yes, get this app back up!" and it wakes in a few seconds — no harm done, and your live Drive connection is unaffected.

If you get stuck

Most first-time problems are one of these five. Find the symptom, apply the fix.

"No key could be detected" / "Could not deserialize key data" / a padding error

Your private_key is mis-pasted. Re-do it with the triple-quote style: real line breaks, starting at -----BEGIN PRIVATE KEY----- and ending at -----END PRIVATE KEY-----. Make sure no backslashes got doubled and you didn't use single quotes.

"File not found," a 404, or an empty/blank result for one item

That item probably isn't shared with the robot, or the ID is wrong. Re-open the item, confirm the robot's email is listed as Viewer, and re-copy the ID from the address bar (only the string after /d/). A folder you forgot to share returns nothing rather than an error — so check sharing first.

A permission / 403 error mentioning the Drive or Sheets API

You likely skipped one of the Enable buttons in Step 1. With your project selected, re-open the Drive API and Sheets API links and confirm both say "API enabled."

The "Create new key" button is greyed out or refuses

Your account is a managed organization (see the Step 1 gotcha). The org blocks downloadable keys by default. Ask an admin to set iam.disableServiceAccountKeyCreation to "Not enforced," or run Step 1 with a personal Gmail account instead.

You changed the secrets but the app still behaves the same

Saved secrets only take effect on the app's next run. Open the app's ⋮ menu in your Streamlit workspace and choose Reboot, then reload the page.

The new Zoom session doesn't show up after you updated the zip

If you deleted and re-uploaded the zip, it has a new ID now. Either upload as a new version of the existing file (keeps the ID), or copy the new ID into attendee_zip_id in Settings → Secrets and re-share the new file with the robot as Viewer.
